# 示例提示词系统实现文档

## 📋 项目概述

本文档记录了为"秒懂AI超级写作员工"项目实现示例提示词系统的完整过程，确保用户在使用各个功能时都能获得清晰的输入指导。

**实现时间：** 2026-01-29
**功能模块：** 动态示例提示词系统
**适用范围：** 所有写作模板

---

## 🎯 需求背景

### 问题描述

用户在使用写作功能时，经常不知道应该输入什么内容，导致：
1. 用户体验不佳，需要反复尝试
2. AI生成的结果质量不稳定
3. 用户学习成本高

### 解决方案

为每个写作模板提供3个具体的示例提示词，帮助用户：
- 快速理解功能的使用方式
- 了解输入内容的格式和详细程度
- 通过示例学习如何描述自己的需求

### 2. 获取示例的工具函数

```typescript
// 获取当前模板的示例提问
const getExamplePrompts = (templateId: string): string[] => {
  return examplePromptsByTemplate[templateId] || examplePromptsByTemplate["default"];
};
```

### 3. 示例索引状态管理

```typescript
const [currentExampleIndex, setCurrentExampleIndex] = useState(0); // 当前示例索引
```

### 4. 插入示例按钮实现

```typescript
<Button
  variant="link"
  size="sm"
  className="text-primary p-0 h-auto"
  onClick={() => {
    const examples = getExamplePrompts(templateId);
    const nextIndex = (currentExampleIndex + 1) % examples.length;
    handleExampleClick(examples[currentExampleIndex]);
    setCurrentExampleIndex(nextIndex);
  }}
>
  插入示例 {currentExampleIndex + 1}/{getExamplePrompts(templateId).length}
</Button>
```

**功能特点：**
- 显示当前示例的序号（如：插入示例 1/3）
- 点击后循环显示下一个示例
- 自动将示例内容填充到输入框

## 📝 示例提示词设计原则

### 1. 具体性原则

❌ **不好的示例：**
```
"帮我写一篇小红书文案"
```

✅ **好的示例：**
```
"我是一名时尚博主，想要创作一篇关于秋季穿搭的小红书笔记，目标受众是25-35岁的都市女性"
```

### 2. 场景化原则

示例应该包含：
- **身份定位**：我是谁（博主、店主、达人等）
- **内容主题**：要创作什么内容
- **目标受众**：面向哪些人群
- **期望效果**：想要达到什么目的

### 3. 多样性原则

每个模板的3个示例应该覆盖：
- 不同的内容领域（时尚、美食、旅行等）
- 不同的用户角色（博主、商家、个人等）
- 不同的创作目的（涨粉、带货、分享等）
